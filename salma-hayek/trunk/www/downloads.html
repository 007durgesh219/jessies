Linux (x86 and x86_64) / Solaris (x86)

For Ubuntu and other Debian-based Linux distributions, there are .i386.deb">x86 (i386) and .amd64.deb">x86_64 (amd64) .deb packages which add to your GNOME applications menu. (For command-line or non-GNOME users, there's /usr/bin/.)

For Fedora and other RPM-based Linux distributions, there are .i386.rpm">x86 (i386) and .x86_64.rpm">x86_64 (amd64) .rpm packages, equivalent to the .deb packages.

For Solaris, there's .i86pc.pkg">SJO.i86pc.pkg for installation with pkgadd(1M), equivalent to the x86 .deb package.

Mac OS (Intel and PowerPC)

For Mac OS 10.4, there's .dmg">.dmg which contains a universal binary application that you can simply drag and drop to install.

MS Windows

For Windows, there's a Windows installer, .msi">.msi, which requires that you've already set up Cygwin (including Cygwin Ruby) yourself but is otherwise complete and leaves you with an icon on your desktop. There are several common Cygwin/Ruby problems.

Other Platforms/Architectures / Building from Source

If no pre-built package is available for your platform, or you just like building from source, anonymous read-only Subversion access is available. 

mkdir ~/software.jessies.org
cd ~/software.jessies.org
svn checkout http://software.jessies.org/svn//trunk/ 
svn checkout http://software.jessies.org/svn/salma-hayek/trunk/ salma-hayek
# You may want to add something like this to your crontab:
#00 23 * * * echo ~/software.jessies.org/salma-hayek/bin/nightly-build.rb ~/software.jessies.org clean build | bash --login

There's also .tar.gz">.tar.gz but Subversion makes it easier for you to keep up to date, and makes it easier to submit patches.

The hints for building are the same for all our projects.
