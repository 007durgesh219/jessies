Downloads

Linux (x86)

For Ubuntu and other Debian-based Linux distributions, there's .deb">.deb which will add to your GNOME applications menu. (For command-line or non-GNOME users, there's /usr/bin/.)

For Fedora and other RPM-based Linux distributions, there's .rpm">.rpm, which is equivalent to the Debian package.

Note that we currently only have packages for the x86 (i386) architecture. For any other architecture, including x86_64 (amd64), you'll have to build from source.

Mac OS (Intel and PowerPC)

For Mac OS 10.4, there's .dmg">.dmg which contains a universal binary application that you can simply drag and drop to install.

MS Windows

For Windows, there's a Windows installer .msi">.msi which requires that you've already installed Cygwin, including Ruby, yourself but is otherwise complete and leaves you with an icon on your desktop.

We have more information about setting up Cygwin.

(Why do we need Cygwin? Mainly for Terminator's benefit. NT's built-in POSIX subsystem won't let you launch native Windows applications or interact with the network, so that wasn't an option. Cygwin, on the other hand, even contains the Unix98 pseudo-terminal support which we use to support terminal resizing, not to mention containing a port of one of the most widely used ssh programs.)

Reporting Bugs

We're working on a proper public bug database, but at the moment, you should use "Report a Bug" from the application's "Help" menu if you're able.

If you're not able (because you're having trouble starting the application, for example), mail us at .

Writing a good bug report

Remember to tell us the application (and version), the OS (and version), the architecture (x86, x86-64, PowerPC) — the "Report a Bug" menu item automatically encodes all this information in the subject line.

A good bug report usually answers these questions: what did you do? What happened? What did you expect would happen? If you know how to make this happen every time, give us the exact steps; we might not be able to guess. You might also want to justify your expectation by telling us which different application or earlier version of our application does what you expected.
